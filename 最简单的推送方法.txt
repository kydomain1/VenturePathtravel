   最简单的GitHub推送方法
========================================

推荐方法：使用 GitHub Desktop（最简单！）
========================================

第1步：下载GitHub Desktop
   访问：https://desktop.github.com/
   下载并安装

第2步：登录
   打开GitHub Desktop
   点击 "Sign in to GitHub.com"
   输入您的GitHub用户名和密码
   (kydomain1)

第3步：添加仓库
   点击菜单 File → Add local repository
   点击 "Choose..."
   选择文件夹：D:\VenturePathtravel
   点击 "Add repository"

第4步：发布到GitHub
   点击顶部的 "Publish repository" 按钮
   
   设置：
   - Name: VenturePathtravel
   - Description: Minimalist lifestyle blog website
   - ✅ Keep this code private (如果想设为私有)
   - 或 ☐ Keep this code private (公开)
   
   点击 "Publish repository"

第5步：完成！
   等待上传完成
   访问查看：https://github.com/kydomain1/VenturePathtravel

备选方法：命令行推送
========================================

如果您更喜欢用命令行：

1. 打开PowerShell
   (按 Win+X，选择 Windows PowerShell)

2. 执行以下命令：

   cd D:\VenturePathtravel
   
   git config --global user.name "kydomain1"
   git config --global user.email "your-email@example.com"
   
   git push -u origin master

3. 输入凭据：
   Username: kydomain1
   Password: [您的Personal Access Token]
   
   ⚠️ 密码必须用Token，不是GitHub密码！
   
   获取Token：https://github.com/settings/tokens
   点击 "Generate new token (classic)"
   勾选 "repo"
   复制生成的token

========================================

如果推送失败：
========================================

错误："refusing to merge unrelated histories"
解决：
   git pull origin master --allow-unrelated-histories
   git push -u origin master

错误："Updates were rejected"
解决：
   git push -u origin master --force
   (⚠️ 会覆盖远程仓库，谨慎使用)

错误："Support for password authentication was removed"
解决：
   必须使用Personal Access Token，不能用密码
